Agent Detail Page
Deep performance view per agent

Where it's accessed

  • A "View Details" button on every agent row in the Agents page.
  • A "View Details" button on every agent row in the Reports page.

What the page shows

  • Profile card — name, designation, contact, CNIC, guardian no, address.
  • How many leads / clients this agent has contacted.
  • Count for every status — each pipeline stage with its own number:
New Contacted Qualified Proposal Negotiation Won (Closed) Lost (Rejected) Dead
  • Grouped totals — Interested, Not-Interested, Qualified, Rejected, and all closings of clients.
  • Month filter — all stats filterable by month so you can review any period.
  • Printable — the agent's full report can be printed professionally.
4
Teams & Sales Managers
New hierarchy in the Leads section

Structure

  • A list of Sales Managers, each leading a team.
  • Teams can be named — Team A, Team B, Team C — each with a team leader and several agents working under them.
  • This lives on its own separate page.

Team statistics (per team, separate page)

  • Response / activity of each team.
  • Sales of each team.
  • Earnings of each team.
  • Drill down from a team to any individual agent's detail page.
5
Accounts Panel
Brand-new module

Salaries & Commissions

  • View each agent's salary and commission.
  • Track whether salary / commission is paid or not.

Expenses & Income

  • Marketing expense, office expense, and other costs.
  • Total income, with date-wise filters throughout.

Client Payments & Immature Files

  • Track client payment details — e.g. down payment of 6 lakh, but file closed at 3 lakh → an immature file.
  • On an immature file, the rebate / agent commission stays pending.
  • See the reason a payment is due — whether we don't have funds, or the file is immature.
Key concept: An immature file holds commission in a "pending" state with a recorded reason, until the file matures or funds are available.

Decisions Needed From You
Answer before we start
1
Commission — Is agent commission a fixed % of the booking amount, or entered manually per deal? This decides how the accounts module calculates earnings.
2
Immature file — What exactly triggers it? Client paying below a set % of the price, or admin manually flagging it? Defines when commission goes "pending".
3
Sales Manager — Can they also work leads like an agent, or only view their team's stats? Affects their login & permissions.
4
Teams — Can an agent belong to multiple teams, or strictly one team only? Affects the team membership design.
5
Phone-only leads — What should display as the "name" when only a phone number exists — "Unknown", or the phone number itself? For lead tables.
6
Client payments — One booking amount only, or full installment tracking? Decides how detailed the payments table is.
